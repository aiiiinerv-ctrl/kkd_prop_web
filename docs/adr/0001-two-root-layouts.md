# Two root layouts instead of one shared `app/layout.tsx`

The public site (`src/app/[locale]/layout.tsx`) and the admin backend (`src/app/admin/layout.tsx`) are separate Next.js root layouts, each with its own `<html lang>`, rather than a single shared root layout with conditional chrome. We chose this because the two surfaces have fundamentally different requirements that would otherwise leak into each other: the public site is bilingual (`NextIntlClientProvider`, `lang` varies by locale) and must be indexed by search engines, while admin is Thai-only, `noindex`, and needs `QueryProvider` + `sonner` instead of the public providers. A shared root layout would need locale-aware conditionals sprinkled through a file that's supposed to be static configuration. The trade-off is duplicated font declarations (Noto Sans + Noto Sans Thai) in both layouts — accepted because the alternative (one layout branching on route) is more surprising to a reader than two small, single-purpose layouts.
